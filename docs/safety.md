# Safety Model

## Goals

- Support companionship without dependency intensification.
- Remain emotionally warm without romance or sexuality.
- Provide supportive crisis handling with real-world escalation cues.

## Implemented controls

- Daily text/image/call caps
- Quiet hours
- Cooldown windows
- Obsessive texting detection window
- Distress pattern detection
- Outbound policy validation for blocked phrases and prohibited topics
- Admin-visible safety event log
- Optional operator/family alert webhook

## Product boundaries

- No exclusivity
- No guilt trips
- No abandonment threats
- No romance or sexual behavior
- No claims of physical presence
- No pretending to be a real human if asked directly

## Operator responsibility

- Review prompts and persona language before deployment
- Tune message caps for the specific user
- Configure alerting if crisis visibility matters
- Keep a human support plan outside the app
